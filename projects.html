Predicting Dementia on 3D Brain MRIs

Alzheimer's Disease (AD) is a debilitating neurological condition characterized by progressive cognitive decline and neurodegeneration. Early detection is crucial for timely intervention and treatment. This project focuses on leveraging advanced deep learning methods to analyze 3D brain Magnetic Resonance Images (MRIs) for the detection of AD, Healthy Control (HC), and the intermediate stage of Mild Cognitive Impairment (MCI).

The methodology involves adapting a sophisticated deep-learning model previously developed for AD classification. Post hoc explanations are incorporated using Layer-wise Relevance Propagation (LRP), enhancing interpretability. The extension of the model to include MCI aims to address the challenge of identifying patients in this critical intermediate stage.

Key findings from the analysis include the identification of diagnostic hallmarks in hippocampal and ventricular brain regions, aligning with established literature. The AD score distribution demonstrates distinct mean values for HC and AD classes, showcasing the model's discriminatory capabilities.

However, the project reveals challenges in multi-class classification, indicating the difficulty distinguishing MCI from HC and AD. The study also suggests future directions, such as incorporating additional risk factors like age to enhance classification accuracy.

In summary, this project contributes to advancing AD detection using deep learning, providing insights into model performance, interpretability, and potential enhancements for more accurate classification in complex MCI scenarios. The combination of advanced AI techniques and a nuanced understanding of model outputs offers promise for improving clinical decision-making in diagnosing neurodegenerative disorders.

Internship Project: Analysis of Intent Recognition for Modular Dialog System

The focus of this internship project is to explore and develop Intent-Based Dialog Systems (IBDS) and address the challenges associated with modular structures in such systems. In practical applications, Dialog Systems (DS) are often designed with a modular approach, combining multiple DS into a unified system known as Modular Dialog Systems (MDS).

The project introduces the concept of Intent-Based Dialog Systems (IBDS), which differ from traditional research-oriented dialog systems by employing rule-based Dialog Managers (DM) in industry applications. The absence of an established term for such systems in the literature leads to the adoption of "Intent-Based Dialog Systems."

The core challenge addressed in the project is the task of module selection (MS) in IBDS. This involves choosing the appropriate sub-DS for each user utterance to generate a unified response. This problem is particularly relevant in industry applications where IBDS, relying on rule-based DM, combines multiple dialog systems with different intents and architectures.

The project contributes significantly to the scientific literature by formally introducing the novel task of MS and providing an evaluation framework for this task. A benchmark dataset is proposed to evaluate models for MS, and three baseline models are presented. The experiments conducted demonstrate the importance of the user utterance text as a more robust feature for MS models compared to confidence values of Intent Detection (ID).

In summary, the internship project makes valuable contributions to advancing the understanding and development of Intent-Based Dialog Systems. It addresses the unique challenges posed by modular structures in practical applications and provides a clear evaluation framework for the task of module selection.

External and Distributed Sorting using Apache Hadoop MapReduce

Shuffling in MapReduce is the process of transferring data from the mappers to reducers1. This phase is necessary for the reducers, otherwise, they would not have any input (or input from every mapper). As shuffling can start even before the map phase has finished, this saves some time and completes the tasks in lesser time1.

Sorting in MapReduce covers the merging and sorting of map outputs. Data from the mapper are grouped by the key, split among reducers, and sorted by the key1. Every reducer obtains all values associated with the same key1. The keys generated by the mapper are automatically sorted by the MapReduce Framework, i.e., before starting of reducer, all intermediate key-value pairs in MapReduce that are generated by mapper get sorted by key and not by value1. Sorting in Hadoop helps reducer to easily distinguish when a new reduce task should start. This saves time for the reducer1.

These two phases occur simultaneously and are done by the MapReduce framework1. This process allows for efficient and distributed processing of large datasets across a cluster of computers, making it a fundamental component of the Hadoop ecosystem.

Overall, “Stream Window Aggregations using Kafka Streams” is a robust solution for real-time data processing and analytics, providing valuable insights and aiding in decision-making processes.

This project is a sophisticated integration of three advanced technologies: Whisper ASR, Facebook’s m2m100 model, and Nvidia’s NeMo. Each of these components plays a crucial role in the pipeline.

OpenAI's Whisper ASR: This is the first step in the pipeline. It takes speech from various sources such as microphones, MP3 files, or live audio streams, and transcribes them into text. This is particularly useful in scenarios like transcribing lectures, meetings, or interviews in real-time.

Facebook’s m2m100 model: The transcribed text is then fed into this machine translation model, which translates the text into a specific language. This is a boon for global communication, breaking down language barriers in international conferences, customer support, or even in social media interactions.

Nvidia’s NeMo: The final step in the pipeline is text-to-speech synthesis. The translated text is converted back into speech, producing audio streams or audio files. This can be used to generate multilingual podcasts, voiceovers for videos, or even for accessibility purposes, aiding those with reading difficulties or visual impairments.

The integration of these three services in a sequential pipeline makes my project a comprehensive solution for multilingual speech processing. PhytoPhinder is an innovative plant disease detection system that harnesses the power of Convolutional Neural Networks (CNNs). The project utilizes a comprehensive dataset of leaf images from Kaggle, known as the Leaf Disease Detection Dataset.

This dataset provides a diverse range of leaf images, covering various species and disease states, which aids in training a robust and versatile model. The dataset comprise with approximate ~90k of images (train ~70k, test ~20k). The dataset only provide 38 types of disease and healthy images of several plants. I hyper-parameter tune it with differnt CNN models (notebook_1.ipynb, and phytophinder-the-leaf-detective.ipynb) with different inner-layers. Validation-loss defines the perforfence of the best model. Where phytophinder-the-leaf-detective.ipynb is the final version of my code. Later I convert it an anssemble everything in train.py.
